OPEN SOURCE@HACKILLINOIS

More about Open Source@HackIllinois

We want hackers to be able to make meaningful addtions to the field of computer science and to organizations that depend on the technology that open source provides. If you do accept working on Open Source, we ask that you be sure:

  1. You are coming to Hackillinois
  2. You want to work on Open Source during the hackathon, knowing there are no prizes
  3. You won't drop Open Source during the hackathon, as you will be letting your team down
We have assembled a varied set of Open Source Projects that hackers may work on.
  • If you have been accepted to Open Source@HackIllinois, let us know you preference. (Note: Not all of these projects necessarly will have a mentor day of based on the prefence of the mentor and the popularity of the project, If you select a project that won't have a mentor during the hackathon we will do our best to put you on a similar project, or you can work on the project without a mentor)

  • If you have have not been accepted to Open Source@HackIllinois but still want to work on Open Source projects, we commend you and we would love for you to do that. Just know that you most likely will not have a mentor hacking with you, though you may still ask for support on the mentorship portal. Also let us know if you are doing so, if a mentor has the time we will direct them to work with you.
You are free to join Open Source after the beginning of the hackathon, you may be added to a existing team or asked to hack with out a mentor (see above to learn how that works)

CoinSorter: Coin grading and Counterfeit Detection

Description: Over the years there have been small changes in the same designs of coins. For example the spacing of “AM” in AMERICA is different on the back of US Lincoln pennies after 1992. Having a large database of coin images, front and back, and knowing their date and mint mark one could correlate regions on both sides of the coin. Most of these will be meaningless, but some correlations of regions will be quite interesting. Using the existing CoinSorter system one will be able to detect rare verities and counterfeit coins already known and quite possibly ones that are unknown.
Technologies: Deep Learning, CUDA, Computer Vision, Hardware Development, Mechatronics
Mentor: Paul Krush
Useful Documentation: https://github.com/GemHunt/CoinSorter/

CoinSorter: Metering, Spacing, and Classification of Coins

Description: Currently CoinSorter only handles one size of coin at time with an off the shelf coin hopper. One should be able to handle mixed coin sizes with a series of independently driven conveyors controlled with an Arduino. Handling mixed coin sizes would make the system much more useful. For example it could find rare coins while doing the simple act of sorting and counting mixed change. This might involve some mechanical changes to conveyor belt guides and fingers or some electronic changes such as adding IR or eddy current sensors.
Technologies: Arduino, OpenCV, C++, Python, SQL, image processing, machine learning, Caffe, convolutional neural networks, basic mechanical/robotics design
Mentor: Paul Krush
Useful Documentation: https://github.com/GemHunt/CoinSorter/

CoinSorter: Port to Android

Description: Currently a Windows PC classifies Caffe models and sends commands to control Arduino based conveyors and solenoids for sorting coins. An Android based system would great option to reduce the cost, size and complexity of the system.
Technologies: Android, Arduino, OpenCV, C++, Python, SQL, image processing, Machine Learning, Caffe, Convolutional Neural Networks
Mentor: Paul Krush
Useful Documentation: https://github.com/GemHunt/CoinSorter

Apache Airavata Allocation Manager

Description: Apache Airavata is software for building science gateways--user environments that help scientists make the best use of supercomputers to solve challenging problems. Airavata follows a microservices architecture. A key missing component in Apache Airavata today is to give science gateway administrators the ability to assign and manage allocations to their users to make sure that a few users don’t use up the gateway’s entire supercomputing allocation. Students who participate in this activity will develop a new Apache Airavata component that will include a database backend, middleware, and an API for the component. The students will also develop form-based submission and approval process that will prospective gateway scientists will use when applying to use a science gateway. Apache Airavata has mentored over 20 Google Summer of Code students since 2013 and is looking for more students in 2016.
Technologies: Java, C++, Python
Mentor: Suresh Marru
Useful Documentation: https://issues.apache.org/jira/browse/AIRAVATA-1893

Description: Apache Airavata is software for building science gateways--user environments that help scientists make the best use of supercomputers to solve challenging problems. Scientific workflows allow scientists to couple two or more applications, perhaps running on supercomputers across the world, into a single composite application. Apache Airavata manages the executions of these workflows and uses JSON to describe a specific workflow.
